# 博客项目实施计划审阅

## 项目概述

构建一个基于 Notion 作为 CMS 的博客网站，使用 Next.js 技术栈部署到 Vercel。

## 需求清单

- 数据源：Notion（通过 Notion API 获取内容）
- 技术栈：Next.js
- 部署平台：Vercel
- 核心功能：
  - 基础博客功能（文章列表、详情页、分类、标签）
  - 全文搜索
  - 评论系统（支持评论和点赞）
  - 暗黑模式
  - 代码高亮

## 模块划分方案

| 模块 | 职责 | 依赖 | 可并行 |
|------|------|------|--------|
| **core** | Notion API集成、数据获取、缓存策略 | 无 | ✅ |
| **ui** | 主题/样式系统、暗黑模式、基础组件库 | 无 | ✅ |
| **blog** | 文章列表页、详情页、分类、标签 | core, ui | ❌ |
| **search** | 全文搜索功能（客户端/服务端） | core, blog | ❌ |
| **comment** | 评论系统 + 点赞功能 | core, blog | ❌ |
| **deploy** | Vercel部署配置、环境变量管理 | 其他全部 | ❌ |

## 技术选型

| 领域 | 选择 | 理由 |
|------|------|------|
| 框架 | Next.js 14+ | App Router, Server Components |
| 样式 | Tailwind CSS | 原子化CSS，暗黑模式支持好 |
| UI组件 | shadcn/ui | 与Tailwind配合，可定制 |
| Notion SDK | notion-client | 官方推荐 |
| 搜索 | Algolia / FlexSearch | 全文搜索方案待确定 |
| 评论 | Giscus / Cusdis / 自建 | 评论系统方案待确定 |
| 部署 | Vercel | Next.js原生支持 |

## 关键问题待讨论

1. **搜索方案选择**：
   - Algolia：功能强但需要第三方服务
   - FlexSearch：纯客户端，适合小博客
   - Meilisearch：自建，有成本

2. **评论系统选择**：
   - Giscus：基于GitHub Discussions，免费
   - Cusdis：轻量级，可自托管
   - 自建：完全可控但工作量大

3. **点赞功能实现**：
   - 需要数据库吗？（Upstash Redis? PlanetScale?）
   - 还是使用第三方服务？

4. **模块划分优化点**：
   - 是否需要将 "core" 拆分为更细粒度？
   - "ui" 模块范围是否过大？
   - "search" 和 "comment" 是否真的可以并行？

## 审阅要点

作为资深架构师，请评审以下内容：

1. **模块划分是否合理？**
   - 每个模块的职责是否清晰？
   - 是否存在功能重叠或遗漏？

2. **模块依赖关系是否正确？**
   - 依赖图是否合理？
   - 是否存在循环依赖风险？

3. **技术选型是否合适？**
   - Next.js + Notion 的最佳实践？
   - 搜索/评论/点赞的最佳方案？

4. **开发顺序是否合理？**
   - 阶段划分是否科学？
   - 是否有更好的并行开发策略？

5. **潜在风险：**
   - Notion API 限制（速率限制、内容大小）
   - Vercel 免费额度限制
   - SSR/SSG 策略选择
